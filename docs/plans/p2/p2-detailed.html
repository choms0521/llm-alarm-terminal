1. 개요

Goal

P1에서 단일 세션으로 검증된 PTY + libghostty 토대를 다중 세션으로 확장합니다. 사이드바 탭 = workspace, workspace 내부에 최대 2개의 가로 분할 pane이 존재합니다. 각 pane은 생성 시점에 claude/shell 중 하나를 선택하며, CHAT_TERMINAL_MAX_SESSIONS (기본 20)이 SessionManager 차원에서 실효성을 가집니다. 첫 번째 탭은 항상 agent-view 종류로 close 불가(placeholder 콘텐츠는 P3에서 실데이터로 교체).

2. RALPLAN-DR Summary

Principles (5)

  1. Build on P1 SessionManager, don't fork the actor model. P1에서 actor isolation을 견고하게 설계한 SessionManager를 그대로 확장합니다. max=1 상수를 max=N으로 풀고 lifecycle hook을 추가하는 surgical extension이며, 별도의 multi-session 매니저를 신설하지 않습니다. P1 → P2의 vertical slice 연속성을 유지합니다.
  2. Workspace as PTY env/cwd isolation boundary. Workspace는 단순한 탭 그룹이 아니라 PTY 환경 격리 경계입니다. 셸 env는 workspace 실행 시점에 snapshot되고, 다른 pane으로 누설되지 않으며, cd/export는 그 pane에 한정됩니다 (수락 기준 A6 핵심).
  3. cmux-style libghostty surface management (ADR-I). 각 세션마다 독립 ghostty_surface_t 인스턴스를 생성하고, hibernate/pause/release 로직을 0줄 작성합니다. 비가시 surface는 PTY read + scrollback + atlas texture를 그대로 유지하며 AppKit displayLayer 호출 부재로만 GPU 자원 idle. 메모리 압박 발생 시 P4+에서 hibernate 재검토.
  4. First tab = agent-view invariant, close-proof. 첫 번째 탭은 kind: .agentView로 자동 생성되고 close 버튼/메뉴 항목/단축키가 모두 차단됩니다. UI 우회 경로(단축키 직접 호출, 메뉴 자동화)도 SessionManager v2 차원에서 거부해야 합니다. P3에서 실데이터로 교체될 때까지 placeholder 콘텐츠로 채워집니다.
  5. Korean-safe persistence with reserved fields. JSON 영속화 스키마는 한국어 workspace 이름을 UTF-8 escape 없이 그대로 저장하며, P5 push 채널 매핑(pushChannelHints), P6a chatRoomId, P10b fetchHint 등 후속 단계의 필드를 reserved 형태로 미리 예약하여 마이그레이션 시 add-only 정책을 보장합니다.

Decision Drivers (Top 3)

  1. 다중 세션 동시성 안전이 본 단계 최대 risk. 20개 PTY가 동시에 read/write할 때 actor isolation 경계 설계가 부정확하면 data race가 발생합니다. P1의 단일 세션 코드에서는 노출되지 않았던 위험이 본 단계에서 전면화됩니다.
  2. JSON 영속화 스키마는 12 phase 전체에 묶임. P5(push 채널), P6a(chatRoomId), P9(chat-room↔pane 매트릭스)에서 새 필드가 추가되며, P2에서 잘못 설계하면 매 단계마다 marshal/unmarshal 호환성을 깨뜨립니다. reserved field + add-only 정책으로 흡수.
  3. libghostty 다중 surface 자원 사용량의 미지수. P1에서 단일 surface는 검증되었으나, 20 surface 동시 운영 시 메모리/GPU 부담은 아직 측정되지 않았습니다. ADR-I cmux 패턴이 채택되었으나 P2 일자에 telemetry hook을 심어 실측 가능 상태로 만들어야 합니다.

Viable Options Considered (3)

옵션설명장점단점판정
A. SessionManager v2 확장 + Workspace as env boundary + cmux surface P1의 SessionManager actor를 max=N으로 확장. Workspace는 env/cwd snapshot의 boundary. 각 세션마다 독립 libghostty surface, hibernate 없음(ADR-I). P1 코드 재사용, vertical slice 연속성, ADR-I 와 정합, env 격리가 데이터 모델에 내장. 20 세션 동시 운영 시 메모리 400~600MB 부담(수용 가능 범위, telemetry로 모니터). 선택
B. Per-workspace SessionManager (sharded actor) Workspace마다 별도의 SessionManager actor를 두어 isolation을 데이터 차원으로 강화. workspace 간 동시성 분리 명확, actor 경계가 작아 추론 용이. P1 SessionManager fork에 해당, max=N 전역 enforcement가 복잡해짐, lastClaudeSessionId 추적 분산. Principle 1(P1 actor model 재사용) 위반. 거부 — P1 actor 모델 fork
C. libghostty surface hibernate / pool 비가시 workspace의 surface를 hibernate하거나 GPU 자원 풀에서 release하여 reactivation 시 재할당. 이론상 메모리 절감 가능. cmux 정탐 결과 hibernate 코드 0줄, scrollback/atlas texture 손실 위험, 재활성화 latency. ADR-I와 정면 충돌. 메모리 압박이 실측되지 않은 상태에서의 조기 최적화. 거부 — ADR-I 위반, 조기 최적화
선택 근거
Option A는 P1과의 연속성(Principle 1), workspace를 env boundary로 보는 acceptance criterion 6(Principle 2), ADR-I cmux 패턴(Principle 3), close-proof agent-view(Principle 4), reserved-field 영속화(Principle 5) 다섯 원칙을 동시에 만족하는 유일한 경로입니다. Option B는 actor fork로 P1의 lastClaudeSessionId/ActivityScope 라이프사이클 추적을 분산시키고, Option C는 ADR-I와 정면 충돌합니다. 20 세션 동시 운영의 메모리 부담은 §6.2의 실측 telemetry로 모니터합니다.

기술 노트
  • 모든 모델은 immutable struct로 정의. 변경은 Workspace.with(...) 빌더로 새 인스턴스 반환(전역 코딩 규약 준수).

기술 노트
  • P1의 actor isolation invariant(모든 public 메서드 async, nonisolated 진입점 0) 그대로 유지. grep 검증을 종료 조건에 재포함.
  • max enforcement: maxSessions는 init 파라미터로 받되 default 값은 ProcessInfo.processInfo.environment["CHAT_TERMINAL_MAX_SESSIONS"] ?? "20"의 parsed Int. 21번째 시도 시 ManagerError.maxSessionsReached(currentMax: 20) throw.
  • env snapshot 시점은 workspace 생성 시점에 ProcessInfo.processInfo.environment를 한 번 캡처하여 Workspace.envSnapshot에 저장합니다 (§5.1, §5.4 captureUserEnv 참조).   • verifier 패스: SessionSpawnEnv 코드에서 ProcessInfo.processInfo.environment 또는 workspace.envSnapshot을 dict copy가 아닌 reference로 전달하는 실수가 없는지, Swift dictionary value semantics가 깨질 가능성이 있는 경로 정탐.

  • Invariant 4 (claude config 격리) 통합 테스트 통과 — 두 workspace의 claude pane이 서로 다른 CLAUDE_CONFIG_DIR 디렉터리 사용 (path 비교)

5.2 SessionManager v2 (max=N, env/cwd snapshot, claude config isolation)

actor SessionManager {
    enum ManagerError: Error {
        case maxSessionsReached(currentMax: Int)
        case notFound(id: UUID)
    }

    private(set) var sessions: [UUID: Session] = [:]

    // H3: P1 max=1 일 때는 `lastClaudeSessionId: String?` (단일 Optional) 이었고,
    // P2 max=N 으로 풀리면서 [sessionId: String] dictionary 로 자연 확장.
    // key 는 sessionId(=Session.id). actor 시그니처/메서드 동일 유지 → fork 아닌 확장.
    private(set) var lastClaudeSessionId: [UUID: String] = [:]
    private var activityScope: ActivityScope? = nil
    private let maxSessions: Int

    init(maxSessions: Int = Self.defaultMax()) {
        self.maxSessions = maxSessions
    }

    static func defaultMax() -> Int {
        let raw = ProcessInfo.processInfo.environment["CHAT_TERMINAL_MAX_SESSIONS"] ?? "20"
        return Int(raw) ?? 20
    }

    func create(workspace: Workspace, paneId: UUID,
                kind: PaneKind,
                rows: UInt16, cols: UInt16) async throws -> Session {
        guard sessions.count < maxSessions else {
            throw ManagerError.maxSessionsReached(currentMax: maxSessions)
        }

        // 1) env snapshot: workspace 생성 시점에 capture 된 envSnapshot 을 base 로 사용 (H6).
        //    workspace 생성 후 user shell rc 변경이 같은 workspace 의 후속 pane spawn 에 영향 없음.
        var envSnapshot = workspace.envSnapshot

        // 2) claude 의 경우 세션별 격리 CLAUDE_CONFIG_DIR 디렉터리 생성
        let sessionId = UUID()
        if kind == .claude {
            let configRoot = try SessionSpawnEnv.claudeConfigDir(forSession: sessionId)
            envSnapshot["CLAUDE_CONFIG_DIR"] = configRoot
        }

        // M3) shell pane: HISTFILE 을 pane 별 격리 디렉터리로 override (Invariant 5).
        if kind == .shell {
            let histDir = try SessionSpawnEnv.zshHistoryDir(
                workspaceId: workspace.id, paneId: paneId)
            envSnapshot["HISTFILE"] = histDir + "/history"
        }

        // 3) command + cwd 결정
        let command: String = kind == .claude
            ? try resolveClaudeBinary()
            : (envSnapshot["SHELL"] ?? "/bin/zsh")
        let cwd: String = workspace.cwd  // 새 pane 의 cwd 는 항상 workspace.cwd (직전 pane 상속 금지)

        let handle = try spawnPTY(
            command: command,
            args: kind == .shell ? ["-l"] : [],
            cwd: cwd,
            env: envSnapshot,   // dict copy by value
            rows: rows, cols: cols
        )

        let session = Session(
            id: sessionId, kind: kind, ptyHandle: handle,
            workspaceId: workspace.id, paneId: paneId,
            cwd: cwd, env: envSnapshot,
            createdAt: Date(), status: .running,
            claudeSessionId: nil
        )
        sessions[session.id] = session

        if activityScope == nil {
            activityScope = ActivityScope(reason: "PTY sessions active")
        }
        return session
    }

    func terminate(id: UUID) async throws {
        guard var session = sessions[id] else { throw ManagerError.notFound(id: id) }
        kill(session.ptyHandle.childPID, SIGTERM)
        try await Task.sleep(nanoseconds: 1_000_000_000)
        if kill(session.ptyHandle.childPID, 0) == 0 {
            kill(session.ptyHandle.childPID, SIGKILL)
        }
        close(session.ptyHandle.masterFD)
        if let claudeId = session.claudeSessionId {
            lastClaudeSessionId[id] = claudeId
        }
        session = Session.with(session, status: .exited)
        sessions[id] = session

        if sessions.allSatisfy({ $0.value.status == .exited }) {
            activityScope = nil
        }
    }

    // M1 + N2: head-of-line stall 회피와 actor reentrancy 양립.
    // 직렬 1초 SIGTERM grace × N panes 대신 withTaskGroup 으로 병렬 wait.
    // N2: child task 가 actor 를 재진입하면 activityScope premature release 위험.
    // 해결: child task 는 actor 진입 없이 kill/waitpid 만 수행. 종료 정리는
    // 모든 child task await 완료 후 actor 안에서 단일 pass 로 수행.
    func terminateAll(inWorkspace workspaceId: UUID) async throws {
        // Step 1 (in-actor): kill 대상 PTY 자원(masterFD, childPID, claudeSessionId) 을 추출.
        let targets: [(UUID, Int32, pid_t, String?)] = sessions
            .filter { $0.value.workspaceId == workspaceId }
            .map { ($0.key, $0.value.ptyHandle.masterFD,
                    $0.value.ptyHandle.childPID, $0.value.claudeSessionId) }

        // Step 2 (out-of-actor): child task 는 actor 재진입 없이 kill/waitpid 만 수행.
        await withTaskGroup(of: UUID.self) { group in
            for (id, masterFD, childPID, _) in targets {
                group.addTask {
                    kill(childPID, SIGTERM)
                    try? await Task.sleep(nanoseconds: 1_000_000_000)
                    if kill(childPID, 0) == 0 {
                        kill(childPID, SIGKILL)
                    }
                    close(masterFD)
                    return id
                }
            }
            // 모든 child task 완료 대기 (reentrancy 발현 0건 보장)
            for await _ in group {}
        }

        // Step 3 (in-actor): 모든 PTY 정리 완료 후 단일 pass 로 상태 갱신 + activityScope 체크.
        for (id, _, _, claudeId) in targets {
            if let claudeId = claudeId { lastClaudeSessionId[id] = claudeId }
            if var s = sessions[id] {
                s = Session.with(s, status: .exited)
                sessions[id] = s
            }
        }
        // Invariant: activityScope == nil iff sessions.allSatisfy { .exited }
        if sessions.allSatisfy({ $0.value.status == .exited }) {
            activityScope = nil
        }
    }
}

5.3 Workspace JSON 영속화 (reserved fields, atomic write)

struct WorkspaceFile: Codable {
    let version: Int                       // 현재 1, 향후 add-only 마이그레이션 anchor
    let workspaces: [Workspace]
    let lastActiveWorkspaceId: UUID?
}

final class WorkspaceStore {
    static let currentSchemaVersion: Int = 1

    private let fileURL: URL
    private let ioQueue = DispatchQueue(label: "workspace.store.io")

    init() throws {
        let appSupport = try FileManager.default.url(
            for: .applicationSupportDirectory, in: .userDomainMask,
            appropriateFor: nil, create: true)
        let dir = appSupport.appendingPathComponent("ClaudeAlarmTerminal", isDirectory: true)
        try FileManager.default.createDirectory(at: dir, withIntermediateDirectories: true)
        self.fileURL = dir.appendingPathComponent("workspaces.json")
    }

    func load() throws -> WorkspaceFile {
        // M4: load 진입부에서 stale .tmp 파일 정리 + .bak 복구 후보 검토
        let tempURL = fileURL.appendingPathExtension("tmp")
        if FileManager.default.fileExists(atPath: tempURL.path) {
            KoreanLogger.warn("이전 저장이 미완료 상태로 종료되었습니다. 임시 파일을 정리합니다.")
            try? FileManager.default.removeItem(at: tempURL)
        }
        guard FileManager.default.fileExists(atPath: fileURL.path) else {
            // .bak 이 존재하면 복구 후보로 검토
            let bakURL = fileURL.appendingPathExtension("bak")
            if FileManager.default.fileExists(atPath: bakURL.path) {
                KoreanLogger.warn("주 파일이 없어 백업(.bak)에서 복구를 시도합니다.")
                try FileManager.default.copyItem(at: bakURL, to: fileURL)
            } else {
                return WorkspaceFile(version: Self.currentSchemaVersion,
                                     workspaces: [], lastActiveWorkspaceId: nil)
            }
        }
        let data = try Data(contentsOf: fileURL)
        let decoder = JSONDecoder()
        decoder.dateDecodingStrategy = .iso8601
        var file = try decoder.decode(WorkspaceFile.self, from: data)

        // MED2: decode 후 agent-view invariant guard. 0개면 생성, 2개+ 면 첫 번째만 유지.
        let agentViews = file.workspaces.filter { $0.kind == .agentView }
        if agentViews.isEmpty {
            file = file.with(prepending: Workspace.makeAgentView())
        } else if agentViews.count > 1 {
            KoreanLogger.warn("agent-view 워크스페이스가 \(agentViews.count)개 발견되었습니다. 첫 번째만 유지합니다.")
            file = file.dedupAgentViews()
        }
        return file
    }

    // H2 + M4: 3-파일 atomic write. workspaces.json (정상) / .tmp (in-progress) / .bak (직전 정상본).
    // replaceItem 실패는 silently swallow 금지 → throws 하여 호출부가 한국어 에러 dialog/log 로 surface.
    func save(_ file: WorkspaceFile) throws {
        let encoder = JSONEncoder()
        encoder.dateEncodingStrategy = .iso8601
        encoder.outputFormatting = [.prettyPrinted, .sortedKeys]
        // JSONEncoder default: 한글을 \uXXXX escape 하지 않음(Swift Foundation 기본 동작).
        let data = try encoder.encode(file)

        let tempURL = fileURL.appendingPathExtension("tmp")
        let bakURL = fileURL.appendingPathExtension("bak")

        // Step 1: write to .tmp
        try data.write(to: tempURL, options: [.atomic])

        // Step 2: 직전 정상 .json → .bak 으로 이동 (존재 시)
        if FileManager.default.fileExists(atPath: fileURL.path) {
            try? FileManager.default.removeItem(at: bakURL)
            try FileManager.default.moveItem(at: fileURL, to: bakURL)
        }

        // Step 3: .tmp → .json (rename atomic)
        try FileManager.default.moveItem(at: tempURL, to: fileURL)
    }
}

5.4 PTY env/cwd 격리 (snapshot at spawn, no propagation)

enum SessionSpawnEnv {
    // H6: workspace 생성 시점에 capture 하는 user env snapshot.
    // Workspace.envSnapshot 에 저장되어 같은 workspace 의 모든 pane spawn 에 사용.
    static func captureUserEnv() -> [String: String] {
        return ProcessInfo.processInfo.environment   // dict value-copy
    }

    // 세션마다 독립 CLAUDE_CONFIG_DIR. 종료 후 디렉터리는 P2 에서 보존(P4 reconnect 활용 예약).
    // M2: 부팅 직후 stale dir cleanup (live sessions 미매칭 + mtime 7일 초과) 별도 수행.
    static func claudeConfigDir(forSession id: UUID) throws -> String {
        let appSupport = try FileManager.default.url(
            for: .applicationSupportDirectory, in: .userDomainMask,
            appropriateFor: nil, create: true)
        let dir = appSupport
            .appendingPathComponent("ClaudeAlarmTerminal", isDirectory: true)
            .appendingPathComponent("claude-config", isDirectory: true)
            .appendingPathComponent(id.uuidString, isDirectory: true)
        try FileManager.default.createDirectory(
            at: dir, withIntermediateDirectories: true)
        return dir.path
    }

    // M3: pane 별 zsh HISTFILE 격리. Invariant 5 (HISTFILE 격리).
    static func zshHistoryDir(workspaceId: UUID, paneId: UUID) throws -> String {
        let caches = try FileManager.default.url(
            for: .cachesDirectory, in: .userDomainMask,
            appropriateFor: nil, create: true)
        let dir = caches
            .appendingPathComponent("ClaudeAlarmTerminal", isDirectory: true)
            .appendingPathComponent("zsh_history", isDirectory: true)
            .appendingPathComponent(workspaceId.uuidString, isDirectory: true)
            .appendingPathComponent(paneId.uuidString, isDirectory: true)
        try FileManager.default.createDirectory(
            at: dir, withIntermediateDirectories: true)
        return dir.path
    }

    // M2: 부팅 직후 1회 실행. live SessionManager.sessions 에 없고 mtime 7일+ 인 dir 삭제.
    static func cleanupStaleClaudeConfigDirs(liveSessionIds: Set<UUID>) throws {
        let appSupport = try FileManager.default.url(
            for: .applicationSupportDirectory, in: .userDomainMask,
            appropriateFor: nil, create: false)
        let root = appSupport
            .appendingPathComponent("ClaudeAlarmTerminal/claude-config", isDirectory: true)
        guard let entries = try? FileManager.default.contentsOfDirectory(
            at: root, includingPropertiesForKeys: [.contentModificationDateKey]) else { return }
        let sevenDaysAgo = Date().addingTimeInterval(-7 * 86400)
        for entry in entries {
            guard let id = UUID(uuidString: entry.lastPathComponent),
                  !liveSessionIds.contains(id),
                  let mtime = try? entry.resourceValues(forKeys: [.contentModificationDateKey])
                                          .contentModificationDate,
                  mtime < sevenDaysAgo else { continue }
            try? FileManager.default.removeItem(at: entry)
        }
    }
}

// PTY env/cwd 격리 invariant 의 5가지 sub-조건 (Day 6 통합 테스트로 검증):
//
 // 1) env snapshot (H6 정정): workspace 생성 시점에 ProcessInfo.environment 를 dict copy
 //    하여 Workspace.envSnapshot 에 저장. SessionManager.create(workspace:) 가 본 snapshot 을
 //    base 로 사용. 같은 workspace 의 모든 pane 은 동일 base env 를 가지며, workspace 생성 후
 //    user shell rc 변경(예: export FOO=BAR)이 같은 workspace 의 후속 pane spawn 에 영향 없음.
 //
 // 2) cd/export 비전파: POSIX 격리에 의해 자연 보장됨. PTY 안의 child 가
 //    수행한 cd/export 는 parent process(= 우리 앱) 에 영향 없음.
 //    다른 pane 에 영향 없음. spawn(2)이 새 child process 를 만들기 때문.
 //
 // 3) 새 pane cwd: SessionManager.create(workspace:) 로 workspace.cwd 를 전달.
 //    직전 pane 의 cwd 를 상속하는 코드 경로가 존재하지 않음 (grep 검증).
 //
 // 4) claude CLAUDE_CONFIG_DIR 격리: 위 claudeConfigDir(forSession:) 가 sessionId 별
 //    독립 디렉터리를 제공하므로 두 claude pane 이 동일 config 를 공유하지 않음.
 //
 // 5) HISTFILE 격리 (M3 신규): shell pane 마다 zshHistoryDir(workspaceId:, paneId:) 가
 //    제공하는 독립 디렉터리에 history 저장. 두 셸 pane 의 명령 기록이 섞이지 않음.

6.3 Trigger for Revisit (P4+ 에서 hibernate 재검토 조건)

중요
ADR-I는 P2 단계의 결정입니다. 다음 trigger 발생 시 P4 이후 단계에서 hibernate/pause/release 정책을 재검토합니다.
  1. 실측 메모리 압박: 16GB+ Mac에서 20 세션 운영 시 메모리 사용량이 1GB를 초과하거나, 사용자 환경에서 "메모리 압박" 시스템 알림이 발생.
  2. GPU 자원 경합: 비가시 surface가 GPU draw call을 발생시키는 회귀가 발견되어 idle 보장이 깨짐.
  3. cmux 회귀: cmux 업스트림이 hibernate 패턴을 도입하면 그 시점에 우리도 alignment 검토.

7.2 Migration Policy (add-only, never break)

  • 버전 증가 규칙: 필드 추가만 허용 → version 유지 가능. 필드 제거/타입 변경/이름 변경 → version 증가 + 마이그레이션 코드 필요.
  • P2 v1 ~ P12 까지의 예상 변화: P5에서 pushChannelHints(workspace level)가 nil→채워짐, P6a에서 chatRoomId(pane level, master § P9 line 351 1:1 바인딩)가 nil→실제 ID, P10b에서 fetchHintMetadata가 활용됨. 모두 reserved field를 채우는 형태이므로 v1 유지 가능.
  • M6: unknown field 보존 정책 (forward-compatible): decode 시 알려지지 않은 필드는 extraFields: [String: AnyCodable]?에 보존됩니다(Workspace/Pane 양쪽). encode 시 그대로 직렬화하여 외부 도구가 추가한 키를 silently drop하지 않습니다. P5+에서 자기 단계의 unknown 필드를 만나더라도 round-trip이 보장됩니다.
  • 마이그레이션 실행 시점: WorkspaceStore.load() 호출 직후 file.version이 current보다 낮으면 migration 함수 체인 호출. P2에서는 v1이 첫 버전이므로 migration 함수가 없으나, hook 지점만 코드에 노출.

7.5 Atomic Write — 3-파일 정책 (M4 격상)

P2 v2에서 단순 atomic write를 3-파일 정책으로 격상합니다: workspaces.json(정상), workspaces.json.tmp(write-in-progress), workspaces.json.bak(직전 정상본).

  1. JSONEncoder로 WorkspaceFile을 Data로 직렬화 (한국어는 escape 없이 UTF-8 그대로).
  2. workspaces.json.tmp에 .atomic 옵션으로 write — POSIX write(2) + fsync(2)에 해당.
  3. 직전 정상 workspaces.json이 존재하면 workspaces.json.bak으로 이동 (rename(2)).
  4. workspaces.json.tmp → workspaces.json rename — POSIX rename(2) atomic semantics.
  5. load() 진입부의 복구 로직 (M4):
    • .tmp 잔존 시: 한국어 warn 로그("이전 저장이 미완료 상태로 종료되었습니다. 임시 파일을 정리합니다.") + .tmp 제거.
    • .json 부재 + .bak 존재 시: 한국어 warn 로그("주 파일이 없어 백업(.bak)에서 복구를 시도합니다.") + .bak → .json 복원.
    • 둘 다 부재 시: 빈 WorkspaceFile 반환 (첫 부팅 케이스).
  6. save() error surface (H2): replaceItem/moveItem 실패는 silently swallow 금지. throws로 호출부에 전파되며 호출부가 한국어 에러 dialog 또는 log로 surface합니다.
