1. 개요

Goal

첫 번째 탭이 모든 workspace의 모든 세션을 카드로 보여 주는 unified Claude Code Agent View. 카드 클릭 시 해당 workspace 탭 + pane으로 점프.

Side

desktop (macOS, Apple Silicon)

  • SessionStatusSnapshot 도메인 모델 (별도 struct, Session 모델 무변경)

2. RALPLAN-DR Summary

Principles (6)

  1. Lifecycle vs Agent-Status 도메인 분리. Session.status (running/exited)는 P1/P2 lifecycle invariant 보존. agent-view용 동적 상태(idle/working/needsInput)는 별도 도메인으로 모델링하여 backward compat 회귀 0건 보장. SessionStatusCoordinator는 SessionLifecycleHooks의 단일 방향 소비자이며, lifecycle hook으로 agent-status를 push하지 않는다 (단방향 invariant).
  2. needsInput 감지는 false negative 허용, false positive 금지. 사용자가 입력 대기 중인데 카드가 idle 표시(FN)는 불편 수준. 입력 대기가 아닌데 needsInput 표시(FP)는 잘못된 액션 유도. v1은 보수적 white-list + SGR reset prefix 조건으로 시작.
  3. 메인 스레드 보호 (이벤트 분리 + 동적 폴링). preview는 throttle 100ms publisher, needsInput은 throttle bypass fast-lane. polling은 focused pane 250ms / background pane 1000ms 동적 빈도로 평균 호출량 절감.
  4. libghostty PTY owning invariant 보존. agent-status 감지가 PTY 재소유 또는 두 번 읽기 금지. ghostty_surface_read_text (viewport snapshot, free 1:1) + ghostty_runtime_action_cb (RING_BELL/PROMPT_TITLE 등 high-level signal, main thread hop) 조합.
  5. UTF-8 메시지 경계 보존. 한국어/ZWJ emoji 포함 preview 추출 시 Data.prefix(byte) 대신 grapheme cluster 단위 truncation. ANSI SGR/CR 파싱도 byte 경계가 아닌 String.Index 단위.
  6. C ABI alloc/free 1:1 강제. ghostty_surface_read_text 호출 시 반드시 defer { ghostty_surface_free_text(...) } 패턴. 메모리 leak 0건이 verifier 4축의 leak/RAII 축에 포함.

D3-2. needsInput 감지 휴리스틱 — versioned policy + telemetry + ❯ FP 강화

// Sources/Session/NeedsInputPolicy.swift (신규)
public protocol NeedsInputPolicy: Sendable {
    var version: String { get }
    func detect(in viewportText: String) -> Bool
}

public struct NeedsInputPolicyV1: NeedsInputPolicy {
    public let version = "v1-2026-05"
    private static let patterns: [String] = [
        "Do you want to apply this change?",
        "Do you want to proceed?",
        "Press Enter to continue",
        "[y/n]",
    ]
    // FP 방지: SGR reset 직후의 ❯ 만 인정
    private static let claudePromptMarker = "\u{1b}[0m❯ "

    public func detect(in viewportText: String) -> Bool {
        let tail = viewportText.suffix(80)
        for pat in Self.patterns where tail.contains(pat) { return true }
        if tail.contains(Self.claudePromptMarker) { return true }
        if tail.contains("\u{1b}[39m❯ ") { return true }
        return false
    }
}

@MainActor
public final class NeedsInputTelemetry: ObservableObject {
    @Published public private(set) var triggerCountThisMonth: Int = 0
    @Published public private(set) var lastTriggeredAt: Date?
    private var monthAnchor: DateComponents?

    /// 월 카운터 reset 조건: 현재 month-of-year != 마지막 record 시점의 month-of-year.
    public func record(now: Date = Date()) {
        let cal = Calendar(identifier: .gregorian)
        let comps = cal.dateComponents([.year, .month], from: now)
        if monthAnchor != comps {
            monthAnchor = comps
            triggerCountThisMonth = 0
        }
        triggerCountThisMonth += 1
        lastTriggeredAt = now
    }
}
  • 패턴은 viewport text 마지막 80 bytes 안에서만 매칭 (스크롤 위쪽 historical 메시지 FP 방지)
  • claude SessionKind == .claude 인 세션에만 적용
  • 셸 세션 working 단순화: "최근 500ms 안에 PTY output(viewport 변경 또는 action_cb) 이 있으면 working, 없으면 idle". 자식 PID/상태 조회는 P4+ deferred.
  • V2 policy bump 시 V1은 namespace 보존 (regression 비교 + telemetry 카운터 추적)

D3-3. PTY output 구독 메커니즘 — 2-trail hybrid

libghostty가 PTY를 owning(P2 invariant). 외부 raw byte 재읽기 금지.

Trail A (high-level signal): ghostty_runtime_action_cb hook. 구독 action:

  • GHOSTTY_ACTION_RING_BELL → needsInput 강신호
  • GHOSTTY_ACTION_COMMAND_FINISHED → working → idle 전이
  • GHOSTTY_ACTION_PROMPT_TITLE → preview 후보
  • GHOSTTY_ACTION_PROGRESS_REPORT → working 강신호
Unknown tag pass-through 정책

4개 known tag 외 미래 신규 tag 등장 시 SessionActionRouter는 pass-through 정책: warn log emit (OSLog category "AgentView.Router") + telemetry counter unknownActionCount 증가 + dispatch 미수행 (vendor bump 호환성).

action_cb는 libghostty read/write/render thread 중 어디에서든 동기 invoke. 따라서 callback 진입 즉시 payload만 추출하고 dispatch는 main thread hop:

// Sources/TerminalView/GhosttyApp.swift 수정 (action_cb 본문)
action_cb: { _, target, action in
    let tag = action.tag
    let surfaceUserdata: UnsafeMutableRawPointer? = {
        if target.tag == GHOSTTY_TARGET_SURFACE,
           let surface = target.target.surface {
            return ghostty_surface_userdata(surface)
        }
        return nil
    }()
    let payload = ActionPayload(extractedFromAction: action)
    DispatchQueue.main.async {
        SessionActionRouter.shared?.dispatch(
            tag: tag,
            surfaceUserdata: surfaceUserdata,
            payload: payload
        )
    }
    return true
}

ActionPayload는 Sendable struct. action union 내 string/integer/enum만 extract 보유. 포인터 보유 금지 (main thread hop 후 libghostty 메모리 invalid 가능).

Trail B (low-level snapshot): viewport polling 동적 빈도. focused pane 4Hz, 그 외 background pane 1Hz.

"focused" 정의

pane이 focused 라는 것은 다음 두 조건 모두 만족하는 상태. (i) manager.selectedID == workspaceId 이고 FocusedPaneStore.focused[workspaceId] == paneId, (ii) NSApp.isActive == true. app 백그라운드 진입 시 NSApp.isActive == false 되어 모든 surface가 background tier (1Hz) 강등. app 활성화 복귀 시 다음 tick에서 focused 재계산.

// Sources/UI/AgentView/ViewportPollingTimer.swift (신규)
@MainActor
public final class ViewportPollingTimer {
    private let focusedIntervalMs: Int = 250
    private let backgroundIntervalMs: Int = 1000
    private var lastPolledAt: [UUID: Date] = [:]
    private weak var focusedPaneStore: FocusedPaneStore?
    private weak var manager: WorkspaceManager?
    private let sessionIndex: SessionIndex
    private let observer: SessionStatusObserver
    private let surfaceRegistry: SurfaceRegistry

    public func tick(now: Date = Date()) {
        let appActive = NSApp.isActive
        let focusedPaneId: UUID? = {
            guard appActive,
                  let wsId = manager?.selectedID,
                  let store = focusedPaneStore else { return nil }
            return store.focused[wsId]
        }()
        for entry in sessionIndex.allRunningSessions() {
            let interval = (entry.paneId == focusedPaneId)
                ? focusedIntervalMs : backgroundIntervalMs
            if shouldPoll(sessionId: entry.sessionId, intervalMs: interval, now: now) {
                poll(entry: entry, now: now)
            }
        }
    }

    /// CRITICAL: ghostty_text_s 의 alloc/free 1:1 강제.
    private func poll(entry: SessionIndexEntry, now: Date) {
        guard let surfaceView = surfaceRegistry.acquireExisting(paneId: entry.paneId),
              let surface = surfaceView.surfaceHandle else { return }
        var text = ghostty_text_s()
        let ok = ghostty_surface_read_text(surface, GHOSTTY_SELECTION_VIEWPORT, &text)
        defer { ghostty_surface_free_text(surface, &text) }
        guard ok else { return }
        let viewportText = String(cString: text.text)
        observer.observe(sessionId: entry.sessionId, viewportText: viewportText, at: now)
        lastPolledAt[entry.sessionId] = now
    }
}

평균 호출량: 1 focused × 4Hz + 19 background × 1Hz = 평균 23Hz (정적 80Hz 대비 71% 절감).

D3-4. SessionStatusCoordinator — 3-publisher 분리

needsInput은 strong signal이므로 throttle 적용 시 D3(사용자 정확성) 충돌. preview/needsInput/status를 별도 publisher로 분리.

// Sources/Session/SessionStatusCoordinator.swift (신규)
@MainActor
public final class SessionStatusCoordinator: ObservableObject {
    @Published public private(set) var snapshots: [UUID: SessionStatusSnapshot] = [:]

    private let previewSubject = PassthroughSubject<(UUID, String), Never>()
    private let needsInputSubject = PassthroughSubject<UUID, Never>()  // throttle 없음
    private let statusSubject = PassthroughSubject<(UUID, AgentStatus), Never>()
    private var cancellables: Set<AnyCancellable> = []

    public init() {
        // (a) preview: 100ms throttle (high-frequency 대응)
        previewSubject
            .throttle(for: .milliseconds(100), scheduler: DispatchQueue.main, latest: true)
            .sink { [weak self] (id, preview) in self?.applyPreview(sessionId: id, preview: preview) }
            .store(in: &cancellables)
        // (b) needsInput fast-lane: throttle bypass, removeDuplicates 만
        needsInputSubject
            .removeDuplicates()
            .receive(on: DispatchQueue.main)
            .sink { [weak self] id in self?.markNeedsInput(sessionId: id) }
            .store(in: &cancellables)
        // (c) status: throttle 없음 (저빈도)
        statusSubject
            .removeDuplicates(by: { $0 == $1 })
            .receive(on: DispatchQueue.main)
            .sink { [weak self] (id, status) in self?.applyStatus(sessionId: id, status: status) }
            .store(in: &cancellables)
    }
    // publishPreview, publishNeedsInput, publishStatus 메서드 + private applyXxx
}

검증 가능 정량:

  • preview 1초 100건 send → applyPreview ≤ 12회 (100ms throttle)
  • needsInput 1건 send → markNeedsInput elapsed < 20ms (fast-lane)
  • status 빈도가 본래 낮아 throttle 불필요

D3-6. AgentJumpAction + FocusedPaneStore + becomeFirstResponder

@MainActor
public final class AgentJumpAction {
    public let manager: WorkspaceManager
    public let focusedPaneStore: FocusedPaneStore
    public let surfaceRegistry: SurfaceRegistry

    public func jump(to snapshot: SessionStatusSnapshot, snapshotIndex: SessionIndex) {
        guard let location = snapshotIndex.locate(sessionId: snapshot.sessionId) else { return }
        manager.select(id: location.workspaceId)
        focusedPaneStore.setFocus(workspaceId: location.workspaceId, paneId: location.paneId)
        if let view = surfaceRegistry.acquireExisting(paneId: location.paneId) {
            view.window?.makeFirstResponder(view)  // GhosttyTerminalView:209-215 호출
        }
    }
}

@MainActor
public final class FocusedPaneStore: ObservableObject {
    @Published public private(set) var focused: [UUID: UUID] = [:]
    public func setFocus(workspaceId: UUID, paneId: UUID) { focused[workspaceId] = paneId }
}

D3-7. 셸 preview — OSC 133 우선 + 3-조건 conjunction fallback

  1. 1순위: OSC 133 marker 존재 시 prompt 영역 직접 식별.
  2. Fallback (OSC 133 marker 부재 시 또는 marker 영역 외부):
    1. SGR + CR 처리 후 줄 단위 분리
    2. 빈 줄, \r로 redraw된 잔여 줄, length≤1 skip
    3. prompt-only 줄 판정 — 3 조건 모두 충족 시:
      • 길이 ≤ 12 grapheme cluster
      • 끝 문자가 % , $ , # , ❯ 중 하나
      • 직전 줄이 빈 줄이거나 CR redraw 직후 (마지막 line 인 경우)
    4. ANSI 잔여 escape 누락 검증: preview에 ESC가 있으면 unit test 실패
    5. 한국어/ZWJ emoji 처리: String.Index + grapheme cluster prefix(200)

RPROMPT 처리: SGR 파서가 CUP/HPA escape 만나면 segment 경계 처리하여 우측 부분 drop.

SGR 파서와 AttributedString 사용은 desktop (macOS) 전용. P4 envelope payload (모바일/WS 서버)는 raw SGR sequence 보존. AttributedString 변환은 데스크톱 카드 렌더 시점에만 적용. 모바일 측 렌더러는 자체 SGR 파서 또는 plain strip 선택권 보유.

CSI 미지원 escape (CUP, ED, HPA)는 segment boundary 처리하되 attribute 미반영. RPROMPT 처리에 활용.

(나) read_text / free_text 1:1 leak invariant

ghostty_surface_read_text(...) 가 채운 ghostty_text_s.text 멤버는 libghostty alloc 메모리. 반드시 동일 surface에 대해 ghostty_surface_free_text(surface, &text) 해제. P3는 ViewportPollingTimer.poll 단일 호출 지점에서 defer 패턴으로 1:1 보장.

verifier 4축 leak/RAII 축 추가 항목:

  • 20 surface × 30분 idle 후 xcrun leaks <pid> count == 0
  • grep -rc 'ghostty_surface_read_text' Sources/ == grep -rc 'ghostty_surface_free_text' Sources/ (vendor 경로 제외)
vendor bump 시 ABI breakage 체크리스트 (시나리오 A 대응)
  • ghostty.h GHOSTTY_API_VERSION 매크로 빌드 타임 assert
  • MemoryLayout<ghostty_text_s>.size 단위 테스트 (member 추가/삭제 detect)
  • defer { free_text } grep 회귀 테스트 1:1 위반 시 CI fail
20 surface ceiling enforcement (시나리오 C 대응)

SessionIndex.size > 20 detect 시 polling 정지 + 한국어 warning toast + telemetry log.

(다) NeedsInput Policy versioning

  • 매 release NeedsInputPolicyV1.patterns trigger rate를 NeedsInputTelemetry.triggerCountThisMonth 로 측정
  • 월 trigger count < 1 시 policy bump (v2) 필요 신호 (claude CLI 출력 카피 변경 의심)
  • v2 bump 시그널 정책 (시나리오 B 대응): V1 monthly counter < 1 1회 발생 시 user-visible 한국어 warn toast + telemetry log
  • v2 release 시 V1은 NeedsInputPolicyV1 namespace 보존 + 기본 정책만 NeedsInputPolicyV2 교체 (regression 비교)
  • FP escalation: 사용자 보고 1건 = HIGH priority bug → patterns에서 즉시 제외 + telemetry counter reset

(라) OSC 133 semantic prompt marker 우선 정책

  • OSC 133 sequence (;A prompt start, ;B command start, ;C output start, ;D command end) 가 viewport text에 1개 이상 존재 시 신뢰
  • 적용 범위: marker 1개 이상 존재 시 해당 marker로 둘러싸인 영역만 OSC 133 우선 정책으로 prompt 영역 추출. marker 외부 텍스트는 fallback heuristic (길이/끝문자/직전 줄 3-조건 conjunction) 처리. viewport가 OSC 133 영역과 raw text 영역 혼재 시 두 정책이 영역별 공존
  • OSC 133 없으면 D3-7 fallback 으로 전체 viewport 처리
  • modern shell (zsh osc133 plugin, fish 3.4+, bash custom PROMPT_COMMAND) 에서만 발행. default zsh 환경에서는 fallback 경로 정상

Soft requirements (외부 PR 의존, 5일 unblock 정책 적용)

  • master § 4 reserved namespace 표에 agentView.sort, agentView.filter 등록 PR 머지 — 책임: Day 6 executor가 master plan PR 작성 + Critic 별도 sub-ralplan 합의 (max 2 사이클)
  • master § 4.3 환경 변수 표에 신규 6개 등록 PR 머지 — 동일 규약
unblock 정책 (5일)

master PR 5일 이상 미머지 시 P3 → P4 진입을 strict block 하지 않는다. 사유:

  1. extraFields["agentView.*"] prefix는 P2 forward-compat catch-all (Workspace.swift M6 decoder)로 master § 4 변경 없이 동작 가능 — P3 기능 자체는 차단되지 않음
  2. P4 진입 시 sub-ralplan 다음 사이클에서 일괄 반영 가능 (P3와 P4가 같은 master 사이클에 묶임)
  3. 환경변수 표는 docstring + 부록 A 보강으로 운영 단계 튜닝 surface 확보

따라서 외부 PR 머지는 "soft requirement"이며 unblock 발동 시 docs/p3/handoff-to-p4.md 에 "master PR debt: 2건" 항목으로 등재. P4 ralplan 첫 사이클의 입력으로 전달.
