## 구현 단계 미결정 (executor 착수 시 확정)

본문 설계에 녹아 있으나 구현 시 최종 확정이 필요한 항목. v2 합의 시점의 open-questions를 이전 보존한다.

- **재부모화 스파이크 결과 (최우선)**: Day 2 GUI 스파이크가 scrollback 보존을 실증하면 정상 경로, 실패하면 read-only 미러 fallback. 단계 설계의 1차 분기. executor가 Day 2 착수 시 최우선 확인. (본문 Day 2 게이트 + Risks 1행)
- **AgentDashboardViewModel 운명**: join 로직이 `AgentTreeBuilder`로 완전 흡수되면 삭제, 부분 흡수면 보존 — Day 1 구현 후 확정. 데이터 인프라는 보존이나 ViewModel 자체는 폐기 후보로 기운다. (본문 Day 3 폐기 절)
- **동시 미러링 UI 힌트 노출 여부**: 워크스페이스 탭에서 세션이 사라질 때 사용자 안내 배지/툴팁을 둘지. designer 청취 항목. P3.5 Q-α-1(사이드바 폭)과 함께 평가. (본문 Risks designer 책임)
- **사이드바 폭 (P3.5 Q-α-1 계승)**: agent-view 활성 시 사이드바 폭 유지(240px) vs 축소. Day 3 walkthrough 결과로 결정.
- **우측 다중 분할**: 동시 2+ 세션 표시를 post-P5.5 토글로 둘지 완전 제외할지. 현재 out of scope이나 각 세션 독립 surface라 기술적으로 가능. 사용자 수요 확인 후 별도 단계 검토. (본문 Scope out)
